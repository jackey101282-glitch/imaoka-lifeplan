## 2. バックアップ手順（ユーザーデータ＝最重要・月1回）

> コードはGitHubに全部あるのでバックアップ不要。**守るべきはSupabaseのユーザーデータ**。

### 方法A：ダッシュボードからCSVで保存（かんたん・推奨）
1. https://supabase.com/dashboard → プロジェクト `imfynopqugqiczevrmsl` を開く
2. 左メニュー **Table Editor** → `app_state` テーブルを開く
3. 右上の **Export** → **Export as CSV** で保存
4. 同様に `members` テーブルもエクスポート
5. ファイル名に日付（例：`app_state_2026-06-22.csv`）を付け、**2人が見れる場所（Googleドライブ等）に保管**

### 方法B：SQLでまとめて取得（確実）
左メニュー **SQL Editor** で実行し、結果をダウンロード：
```sql
select household_id, data, updated_at from app_state;
select household_id, user_id, role from members;
```

### 頻度の目安
- **今（少人数）：月1回**で十分。
- ユーザーが増えてきたら週1回、または有料プランの**Point-in-Time Recovery（自動巻き戻し）**を検討。

## 3. 復旧手順（シナリオ別）

### A. ユーザーデータを誤って消した／壊した
1. 直近のバックアップCSV（§2）を用意
2. Supabase **SQL Editor** で該当世帯の `data` を上書き：
   ```sql
   update app_state
   set data = '<バックアップのJSON>'::jsonb, updated_at = now()
   where household_id = '<対象のhousehold_id>';
   ```
3. 対象ユーザーにアプリの再読込（⌘+Shift+R／アプリ再起動）を案内

### B. pushでアプリが壊れた（最優先で戻す）
```bash
cd "/Users/yuseiimaoka/今岡家ライフプラン管理"
git log --oneline -5            # 直前の正常コミットを確認
git revert HEAD                 # 壊したコミットを打ち消す（履歴は残る）
git push origin main            # 本番が前の正常状態に戻る
```
- それでもダメなら安定版タグへ：`git checkout test-1 -- index.html sw.js && git commit -m "緊急: 安定版へ" && git push`
- push後に必ず `gh api repos/jackey101282-glitch/imaoka-lifeplan/pages/builds/latest --jq .status` が `built` を確認

### C. ドメイン(mietta.jp)が落ちた
1. `dig +short mietta.jp A` を確認 → `185.199.x.x`（GitHub）でなければDNS事故
2. お名前.com Navi でDNS設定を確認（Aレコード4本＝185.199.108〜111.153／NS＝dnsv.jp）
3. 有効期限切れなら更新（自動更新ONだがクレカ期限切れに注意）
4. **暫定**：`https://jackey101282-glitch.github.io/imaoka-lifeplan/` は常に生きてるので、そちらを案内すれば即復旧

### D. Supabaseの鍵が漏れた疑い
1. ダッシュボード → Project Settings → API → **キーをローテーション**
2. `index.html` の `APP_CONFIG` の publishable key を新しいものに差し替え→push
（※publishable keyは公開前提なので漏れても被害は限定的。secret keyは元からクライアントに入れていない）

## 4. 監視（落ちたら自動で気づく）※社長のアカウント操作が必要

無料の **Uptime Robot** で死活監視（各5分間隔・無料50件まで）：
1. https://uptimerobot.com で無料登録
2. **Add New Monitor** → タイプ`HTTPS`
3. 監視URLを2つ登録：
   - `https://mietta.jp/`（本番アプリ）
   - `https://mietta.jp/lp/`（LP）
4. 通知先にメール（or LINE/Slack）を設定 → **落ちたらすぐ通知が来る**

> これを入れておくと「ユーザーの離脱で気づく」のではなく「落ちた瞬間に気づく」状態になる。設定は15分ほど。**アカウント登録だけ社長、設定手順は上記の通り。**
