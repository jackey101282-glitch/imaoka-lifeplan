# Mietta プッシュ通知 設計（2026-07-07 社長FB反映）

対象＝iOSネイティブ(Capacitor + APNs)。パートナー連動はサーバー(Supabase Edge Function→APNs)。設定UIで種別ごとON/OFF＋おやすみ時間。既定は"うるさくない"寄り。

## A. パートナー連動（相手が入れたら通知）
| 種別 | 文面例 | 既定 |
|---|---|---|
| 高額支出の承認リクエスト | 〇〇が ¥50,000 の承認を求めています | ON |
| タスクを依頼／完了 | 〇〇が『保育園の書類』を頼みました／完了しました | ON |
| 予定を追加 | 〇〇が『美容室 7/10』を追加しました | ON |
| **支出を記録** | （まとめ通知に集約。個別即時はしない） | → 下のC-1 |
| やりたい/欲しいもの追加・共有メモ更新・棚卸し | 〇〇が『沖縄旅行』を追加 等 | OFF（任意） |

## B. リマインド（アプリらしさ＝4.2審査対策にも効く）
- 予定の前日/当日／タスク期限／カード締め日(変動費入力促し)／トライアル終了3日前／使いすぎ80%アラート

## C. 社長追加アイデア（2026-07-07・重要）
- **C-1 支出デイリーダイジェスト（1日1回まとめ）**：夜に「今日の支出：食費¥3,200・日用品¥800（計¥4,000）。今月 使えるお金 残り¥X」。※個別の支出即時通知はやめ、これに集約＝通知疲れ回避。
- **C-2 0円デーの記録忘れナッジ**：その日の支出記録が0件なら夜に「今日は支出の記録がありません。使わなかった？入れ忘れてない？」→ 習慣化を促す。0円が続く=離脱サインの検知にも。
- **C-3 予定の実績照合（Mietta独自の目玉）**：金額付きの予定（例『美容室』¥10,000）の日が過ぎたら「『美容室』は予定¥10,000でした。実際いくらでしたか？」→ タップで支出記録に直行。**予定(見込み) vs 実際**の差をあとで見える化でき、「使えるお金」の精度が上がる。
  - 実装メモ：予定は既に金額(`eventAmountForMonth`/`amount`)を持つ。予定日の翌朝などにその予定に紐づく実支出が未記録なら通知。記録済みなら差額を軽く提示。

## 設計方針
- 種別ごとON/OFF＋「まとめて通知」＋おやすみ時間（深夜は送らない）。
- パートナー連動＝Supabase側で相手のデバイストークンへAPNs送信（メンバーごとにpushトークン保存）。
- C-1/C-2/C-3のスケジュール通知＝ローカル通知(端末側)でも実装可（サーバー不要な分は端末で）。パートナー連動(A)だけサーバー必須。

## 実装ブロック
- Block A：設定UI＋通知したい"場面"の判定ロジック（JS側）。
- Block B：社長がAPNs認証キー(.p8)作成→私に渡す。
- Block C：私がプッシュ受信＋サーバー(Edge Function)結線。C-1〜C-3のスケジューリング。
